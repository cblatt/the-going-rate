    How this was built

    166,007 listings collected · 56,498 sorted into 99 comparable groups · 820 priceable combinations · top 1,000 bargains independently audited, 113 removed · built with Claude Code.

    Problem → solution

    Problem: used-guitar prices are folklore. The 166,000 live Reverb listings hold the real answer, but no buyer can read them all.

    Solution: read them all. Sort every guitar into a group of true comparables, price it against its own group with plain percentiles, and surface the verdicts — a price checker, a market overview, a ranked deal feed.

    1 · Collect the data

    • 166,007 listings pulled from Reverb's public API into SQLite — Python stdlib only, nothing to install.
    • One search can only return its first 20,000 results — found by testing, not in the docs — and the market holds 166,000. Fix: pull the market in price ranges ($0–500, $500–1,000, …), splitting any range that's still too big in half, until every listing is reachable.
    • Asking prices, not sale prices — Reverb doesn't publish transactions, so every verdict is relative to today's market.
    • 166,007 collected, 56,498 priced. A guitar is only priced if its listing can be confidently matched to a group of 30+ true comparables — the rest (rare models, small brands, vague listings) are left out rather than guessed at.

    2 · Analyze

    • Value often hides in the seller's own words ("player grade," "as-is," "partscaster"), so the analysis starts by reading them: ~100 hand-written keyword rules sort 56,498 listings into 99 families of true comparables.
    • Every guitar is priced against its own group — same model family, plus era and condition where the group is deep enough, never fewer than 30 comparables. The verdict is a plain percentile: "cheaper than 82% of comparable Strats."
    • Calling something a deal has a higher bar: it must undercut even the cheap quartile of its group, with no damage admitted in the title — so budget models and beat-up guitars can't fake a discount.
    • Final check: a fresh Claude instance re-read the top 1,000 bargains and removed 113 that were filed in the wrong group — e.g. "Custom Shop pickups" is not a Custom Shop guitar.

    3 · Build the site

    • One analysis script precomputes everything into static JSON; the site is plain HTML/JS/SVG — no frameworks, no servers, no keys, no runtime moving parts.
    • Structured around one question — is this guitar priced fairly? — verdict first, evidence below it, deals as the action.

    Where the AI was — and wasn't

    • Every line of code written in a Claude Code session; every judgment call human.
    • The LLM sits only where errors are expensive: auditing the top of the deal feed. Transparent rules for the bulk, intelligence for the edge cases.

    Limits, stated

    • Asking ≠ sold. Validated the one way the data allows: listings this site calls overpriced measurably sit unsold longer.
    • One snapshot, not a time series.
    • Sorting is ~89% accurate on its hardest slice — measured against an independent audit, not assumed.
    • Rare models and small brands stay unpriced: four comparables is no comparison.

    The data & methodology

    Every listing behind the numbers — one pull of Reverb's public listings API, sorted into groups by ~100 plain keyword rules, scored against its own group. Percentile 0.10 = cheaper than 90% of its comparables. Download the full CSV.
